## 三、关键发现

### 3.1 基线合理性验证

- **NoMemory（Score=0.0031）**：几乎全军覆没，仅有 1 题 PARTIAL。验证了 LoCoMo 评测集的题目**确实需要对话历史才能回答**，有效下界成立。
- **FullContext（Score=0.2219）**：将全部对话塞入 prompt 在 multi_hop 和 open_domain 上有一定效果，但受限于 8000 token 的截断窗口，较早的对话信息被丢弃。
- **VanillaRAG（Score=0.1625）**：向量检索在 open_domain 上意外最强（Score=0.45），但 multi_hop 几乎无效（Score=0.0125），说明纯语义相似度无法处理需要多步推理的问题。

### 3.2 MemoryAgent 的优势

| 维度 | 表现 |
|------|------|
| **single_hop**（Score=0.3125） | 领先所有基线 92% 以上，记忆检索有效定位了单步答案 |
| **temporal**（Score=0.2375） | 领先 FullContext **171%**，Recency 衰减 + 时间感知写入明显起作用 |
| **F1 翻倍**（0.175 vs 0.087） | 说明回答包含更多正确关键词，不仅仅是语义相关 |
| **EM 非零**（0.0563） | 唯一能精确复现参考答案的系统（9 题完全一致） |

### 3.3 MemoryAgent 的短板

- **multi_hop（Score=0.1125）**：落后 FullContext（0.2375）。多跳推理需要跨 session 整合信息，当前检索机制未能有效跨越多个不相关的记忆片段。这是后续重点优化方向。
- **open_domain（Score=0.3875）**：略落后 VanillaRAG（0.45），但差距在可接受范围。

### 3.4 延迟表现

| 系统 | 延迟 | 说明 |
|------|-----:|------|
| NoMemory | 0.133s | 空 ingest + 短 prompt，接近 LLM 裸推理耗时 |
| VanillaRAG | 0.355s | embedding 检索增加约 0.2s 开销 |
| **MemoryAgent** | **0.733s** | 记忆写入、检索、反思等全链路，仍远低于用户容忍阈值 |
| FullContext | 3.414s | 超长 prompt（~8000 tokens）导致 prefill 阶段极慢 |

> MemoryAgent 在 FullContext 的 **21% 响应时间**内达到了 **18% 更高的 Score**，性价比显著。

## 五、后续优化方向

| 优先级 | 方向 | 预期提升 |
|:---:|------|:---:|
| 🔴 高 | multi_hop 多跳检索（跨 session 链式推理） | Score +10~15pp |
| 🟡 中 | open_domain 结合全局摘要与局部检索 | Score +5~8pp |
| 🟢 低 | 反思机制调参（触发阈值 / 频率） | Score +2~3pp |
| 🟢 低 | embedding 模型升级（bge-small → bge-base 或 multilingual） | Score +3~5pp |
