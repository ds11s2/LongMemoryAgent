## 二、环境配置

### 2.1 安装依赖

**以下全部必须装，缺一不可**（在项目根目录执行）：

```powershell
pip install -r eval_kit\eval_kit\requirements.txt
pip install chromadb python-dotenv vllm langchain-openai langchain-core
```

> `requirements.txt` 包含：`openai`、`numpy`、`sentence-transformers`

**安装后必须验证**，逐行运行确认没有 `ModuleNotFoundError`：

```powershell
python -c "import openai; print('openai OK')"
python -c "import numpy; print('numpy OK')"
python -c "import sentence_transformers; print('sentence-transformers OK')"
python -c "import chromadb; print('chromadb OK')"
python -c "import dotenv; print('python-dotenv OK')"
```

> 如果 `sentence-transformers` 安装失败（常见于 Windows，通常都会失败），尝试：
>
> ```powershell
> pip install sentence-transformers --no-cache-dir
> ```

### 2.2 配置 .env 文件

编辑项目根目录的 `.env`。**二选一**：

**方案 A：全云端 API（推荐，无需 GPU）**

如果你没有 GPU 或不想部署本地模型，直接用 DeepSeek 云端 API 跑所有步骤（Generation + Judge）：

```env
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_API_KEY=sk-你的key
LLM_MODEL=deepseek-v4-flash
```

> DeepSeek API Key 获取：<https://platform.deepseek.com>

**方案 B：本地 vLLM（需要 NVIDIA GPU，≥8G 显存）**

```env
LLM_BASE_URL=http://localhost:8000/v1
LLM_API_KEY=EMPTY
LLM_MODEL=Qwen/Qwen2.5-3B-Instruct-AWQ
```

> 选择方案 B 还需额外完成下面的 2.3 节（启动 vLLM 服务）。
>
> `.env` 已在 `.gitignore` 中，不会被提交到 git。每人使用自己的配置。

### 2.3 启动本地 vLLM 服务（仅方案 B 需要）

> 如果你选了方案 A（全云端 API），**跳过本节**，直接看 2.4。

在**新的终端窗口**中运行（不要关闭）：

```powershell
vllm serve Qwen/Qwen2.5-3B-Instruct-AWQ --port 8000 --max-model-len 8192 --gpu-memory-utilization 0.75
```

首次启动会自动从 HuggingFace 下载模型（约 6 GB）。国内网络慢可以设镜像：

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
# 然后再启动 vLLM
```

启动成功后终端会显示 `Uvicorn running on http://0.0.0.0:8000`。

### 2.4 准备评测数据

```powershell
cd eval_kit\eval_kit
python prepare_eval_set.py --output eval_set.json --per_category 10 --seed 42
cd ..\..
```

| 参数               | 作用           | 调试用   | 正式用 |
| ---------------- | ------------ | ----- | --- |
| `--per_category` | 每类题目数（共 4 类） | 5\~10 | 40  |
| `--seed`         | 随机种子         | 42    | 42  |

data 会自动从 GitHub 下载到 `.locomo_cache/`，只需下载一次。

## 三、验证环境：先跑 FullContextAgent 基线

> ⚠️ **这是最关键的验证步骤，必须跑通才能继续**。用最简单的 FullContextAgent 验证依赖、.env、LLM 连通性全部正常。

```powershell
cd eval_kit\eval_kit
python run_generation.py --eval_set eval_set.json --agent agent_template:FullContextAgent --output predictions_check.json --limit_conversations 2
cd ..\..
```

**预期输出**：终端打印 `[2/2] xxx：正在 ingest ...`，无任何 Traceback，完成后生成 `predictions_check.json`（文件约几 KB）。

如果这一步报错，对照下表排查：

| 错误信息                                         | 原因         | 解决                                                          |
| -------------------------------------------- | ---------- | ----------------------------------------------------------- |
| `ModuleNotFoundError: ...`                   | 依赖没装全      | 回到 2.1 节逐条跑验证命令                                             |
| `ConnectionError / 404 / Connection refused` | LLM 连不上    | 方案 A：检查 `.env` 的 `LLM_API_KEY` 是否正确；方案 B：检查 vLLM 是否在另一个终端运行 |
| `openai.AuthenticationError: 401`            | API Key 无效 | 重新生成 key 并更新 `.env`                                         |
| `FileNotFoundError: eval_set.json`           | 评测集未生成     | 回到 2.4 节执行 `prepare_eval_set.py`                            |

**跑通后**，删除测试文件 `del predictions_check.json`，继续下一步。

### 4.1 快速调试（只跑 3 段对话，验证整条链路）

```powershell
cd memory_agent\eval
python run_eval.py --limit_conversations 3 --eval_set ..\..\eval_kit\eval_kit\eval_set.json --output predictions_debug.json --skip_judge
```

| 参数                        | 作用                         |
| ------------------------- | -------------------------- |
| `--limit_conversations 3` | 只跑前 3 段对话                  |

## 五、调整 Settings 参数

所有可调参数在 `memory_agent\agent\controller.py` 的 `Settings` 类中：

| 参数                           | 默认值   | 含义                    |
| ---------------------------- | ----- | --------------------- |
| `retrieval_top_k`            | 5     | answer 时检索几条记忆        |
| `decay_factor`               | 0.995 | Recency 衰减系数，越小近期优势越弱 |
| `reflection_threshold`       | 150   | 累计重要性超此值触发反思          |
| `reflection_memory_limit`    | 100   | 反思取最近多少条记忆            |
| `reflection_question_count`  | 3     | 反思生成几个问题              |
| `reflection_insight_per_q`   | 3     | 每个问题几条洞察              |
| `reflection_retrieval_top_k` | 20    | 反思检索 top\_k           |
| `reflection_max_tokens`      | 256   | 反思 LLM max\_tokens    |
| `reflection_temperature`     | 0.2   | 反思 LLM 随机性            |

修改方式：直接编辑 `Settings` 类中的默认值。改完运行评测即可看到效果。

## 七、常见问题

| 现象                                           | 可能原因                    | 解决                                                                |
| -------------------------------------------- | ----------------------- | ----------------------------------------------------------------- |
| `ModuleNotFoundError: sentence_transformers` | 没装或安装失败                 | `pip install sentence-transformers --no-cache-dir`                |
| `ModuleNotFoundError: chromadb`              | 没装                      | `pip install chromadb`                                            |
| `ModuleNotFoundError: openai`                | 没装                      | `pip install openai`                                              |
| `ModuleNotFoundError: dotenv`                | 没装                      | `pip install python-dotenv`                                       |
| vLLM 连不上                                     | 服务挂了或端口不对               | 确认另一个终端 vLLM 在运行，确认 `.env` 端口是 8000                               |
| 显存不足 (OOM)                                   | 3B 模型 + embedding 模型一起占 | vLLM 加 `--gpu-memory-utilization 0.65`                            |
| Judge 报 401 / AuthenticationError            | API Key 没设或过期           | 检查 `.env` 或 `--judge_api_key`，去 platform.deepseek.com 重新生成        |
| `_parse_date_time` 报错                        | 日期格式变了                  | 检查 eval\_set.json 里的 date\_time 格式是否还是 `"1:56 pm on 8 May, 2023"` |
| ChromaDB 报错                                  | 上次运行的残留数据               | 删除 `chroma_db/` 文件夹重跑                                             |
| Embedding 模型下载慢                              | HuggingFace 被墙          | `$env:HF_ENDPOINT = "https://hf-mirror.com"` 后重跑                  |
| FullContextAgent 能跑但 MemoryAgent 不行          | Agent 代码有 bug           | 先用 `--limit_conversations 1` 单对话调试，看具体哪步报错                        |

> **万能调试法**：任何问题都先退回第三节跑 FullContextAgent。如果基线能跑通，问题在 `memory_agent/` 自身；如果基线也跑不通，问题在环境 / 依赖 / LLM 连通性。

## 八、云端容器环境配置注意事项

> 如果你在算力租用平台（AutoDL / 矩池云等）的容器中配置环境，以下经验可以帮你节省大量时间。

### 8.1 conda base 环境与 .venv 虚拟环境的冲突

容器通常会自带一个 conda 环境（`/opt/conda/`），终端提示符显示 `(base)`。如果你创建了 `.venv` 并 `source .venv/bin/activate`，提示符会变成 `(.venv) (base)`，此时**两个环境同时激活**，conda 的 `/opt/conda/bin` 在 PATH 中优先级更高，导致 `vllm` 等命令实际调用的是 base 里的旧版本。

**解决办法**（在新终端中执行）：

```bash
conda deactivate                    # 先退出 base
conda config --set auto_activate_base false   # 永久禁用自动激活
source .venv/bin/activate           # 激活自己的虚拟环境
```

验证命令确实来自 `.venv`：

```bash
which vllm     # 应输出 /home/.../.venv/bin/vllm，而非 /opt/conda/bin/vllm
which python   # 应输出 /home/.../.venv/bin/python
```

### 8.2 NVIDIA 驱动与 PyTorch CUDA 版本不匹配

**场景**：容器 GPU 为 Tesla T4，驱动 `570.124.04`，最高支持 **CUDA 12.8**。但容器自带 PyTorch 是 `cu130`（CUDA 13.0），导致 vLLM 报错：`The NVIDIA driver on your system is too old`。

**诊断命令**：

```bash
nvidia-smi                          # 看驱动版本和 CUDA Version
python -c "import torch; print(torch.__version__, torch.version.cuda)"
```

**已验证可用的版本组合**（Tesla T4 + 驱动 570）：

| 组件 | 版本 | 说明 |
|------|------|------|
| PyTorch | `2.7.1+cu128` | 必须匹配驱动 CUDA 12.8 |
| vLLM | `0.7.3` | 兼容 torch 2.7 |
| transformers | `4.45.2` | **关键**：0.7.3 的兼容窗口极窄 |

安装命令：

```bash
# 如果容器已有不兼容的 torch，先卸载
pip uninstall -y torch torchvision torchaudio vllm

# 装 PyTorch cu128 版
pip install torch==2.7.1 --index-url https://download.pytorch.org/whl/cu128

# 装 vLLM
pip install vllm==0.7.3

# 装特定版本 transformers（必须在 4.45.x ~ 4.47.x 之间）
pip install "transformers==4.45.2"
```

### 8.3 transformers 版本兼容窗口

vLLM 0.7.3 对 transformers 版本要求非常苛刻：

| transformers 版本 | 结果 |
|---|---|
| 太新（≥4.48） | 报错 `Qwen2Tokenizer has no attribute all_special_tokens_extended` |
| 太老（≤4.44） | 报错 `No module named 'transformers.models.mllama'` |
| **4.45.0 ~ 4.47.x** | ✅ 可用，推荐 `4.45.2` |

> 如果遇到报错说某个属性/模块不存在，不要盲目大幅升降 transformers，微调 0.1 个版本号逐个试。

### 8.4 模型下载（国内网络）

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download Qwen/Qwen2.5-3B-Instruct-AWQ
```

下载完成后 `vllm serve` 会直接使用缓存，无需再次下载。

### 8.5 `.venv` 环境验证清单

启动 vLLM 前，逐条确认：

```bash
conda deactivate                    # 确保没有 (base) 前缀
source .venv/bin/activate           # 激活虚拟环境
python -c "import torch; print('CUDA:', torch.cuda.is_available())"    # 必须 True
python -c "import vllm; print('vLLM:', vllm.__version__)"              # 必须 0.7.3
python -c "import transformers; print('transformers:', transformers.__version__)"  # 必须 4.45.2
which vllm                          # 必须指向 .venv/bin/vllm
```

全部通过后再执行 `vllm serve`。

```powershell

python run_generation.py --eval_set eval_set.json --agent agent_template:VanillaRAGAgent --output predictions_rag.json
cd ..\..
```
